Garage Lens Reader API

Read-only JSON API for Garage Lens over a self-hosted TeslaMate PostgreSQL database. The mobile app only needs a Reader API URL and an access token; PostgreSQL credentials stay on the owner’s server.

Version 0.4.4 Read-only Bearer token

Base URL

https://reader.example.com

Use the HTTPS domain provided by the TeslaMate server owner.

Authentication

Authorization: Bearer <access-token>

/api/health is public health; data endpoints require the token.

App Validation

GET /api/health then GET /api/cars

Garage Lens hides all vehicle data until both checks pass.

Security Notes

Do not expose PostgreSQL to the public internet. Run Reader beside TeslaMate, use a read-only PostgreSQL user, keep HTTPS enabled, and rotate the Reader token after public testing.

Quick Start

curl https://reader.example.com/api/health

curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://reader.example.com/api/cars

curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://reader.example.com/api/cars/1/overview

Overview Shape

vehicle

Display name, model, state, battery, range, odometer, location, temperature, latest sample.

monthlyStats

Distance, drive count, energy, charge energy, efficiency, charge cost, online/asleep hours.

drives

Recent drive cards with start/end, distance, energy, speed, battery levels.

charges

Recent charging sessions with battery delta, kWh, range added, cost, peak power.

database

Connection status, row counts, schema version, database size, reader version.

analytics

Charging costs, battery capacity/degradation, state timeline, mileage, charging curves, speed/temperature efficiency, top stations.

Raw Tables

Reader automatically exposes vehicle-related tables only. This includes direct car_id tables like positions, drives, states, charging_processes; charges through charging_processes.car_id; referenced addresses and geofences; the selected cars row; and compatible future TeslaMate tables.
